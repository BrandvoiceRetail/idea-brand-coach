A pour-over-dripper founder was about to set next year's creative budget the way most founders do: by memory. Which rewrite felt like the win, which agency felt worth renewing, which version everyone on the team still talks about in the group chat.

Memory is a bad budgeting tool. It remembers the version that was easiest to talk about, not the version that actually moved the number.

`get_experiment_lift` and `compute_trust_gap_lift` exist for exactly this moment — the point where you stop guessing which change worked and pull the actual delta. `get_experiment_lift` shows what a specific test did to conversion, before and after, tied to the listing-copy position it ran against. `compute_trust_gap_lift` shows whether the underlying IDEA score — the trust gap itself — actually closed, or whether conversion moved for some other reason entirely and the trust gap is still sitting where it was.

For the dripper founder, the rewrite everyone remembered fondly barely moved the trust gap at all — it read better, but the Empathetic score behind it was flat. The rewrite nobody talked much about, done quietly in March by a freelancer, had closed the biggest gap of the year. That's the one that deserves next year's budget. Not the one with the better internal story.

This is the difference between "creative that felt good" and "creative that worked" — and at year-end, with a full twelve months of tests behind you, it's the cheapest audit you'll run all quarter. You already paid for the tests. All that's left is reading them honestly.

The action: before you write next year's creative budget, pull `get_experiment_lift` on every listing-copy test you ran this year and cross-check the winners against `compute_trust_gap_lift`. Fund what closed the gap. Let the rest go, no matter how good it felt to ship.
